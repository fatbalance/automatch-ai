3.1 ANALYSIS
================================================================================

--------------------------------------------------------------------------------
3.1.1 Problem Definition
--------------------------------------------------------------------------------

The problem: Car buyers in the UK struggle to find suitable vehicles. They must
consider budget (monthly payments vs total cost), fuel economy, insurance
groups, reliability, boot space, and personal preferences. Existing car sites
(e.g. Auto Trader, CarWow) show large lists but offer little personalised
guidance. Users often feel overwhelmed and unsure which car best fits their
needs.

Proposed solution: A web application that collects user preferences, uses
artificial intelligence to recommend suitable cars with explanations, allows
conversational refinement via chat, and guides users to a final choice through
a structured comparison with real specs.

--------------------------------------------------------------------------------
3.1.2 Stakeholder Identification
--------------------------------------------------------------------------------

Primary stakeholder: UK car buyers (first-time buyers, families, commuters)
- Need: Clear, personalised recommendations within budget
- Need: Ability to compare cars side-by-side with key specs

Secondary stakeholder: The developer (student)
- Need: Project that demonstrates programming skills and meets NEA criteria

--------------------------------------------------------------------------------
3.1.3 Computational Methods and Existing Solutions
--------------------------------------------------------------------------------

Computational methods used:
- API integration (REST, JSON) to communicate with external AI service
- Client-side state management (JavaScript variables, localStorage)
- Pattern matching (regular expressions) to extract car names from AI text
- Algorithm: Tournament-style elimination (pairing, winner selection, recursion)

Existing solutions:
- Auto Trader: Search filters, no AI recommendations
- CarWow: Dealer quotes, no conversational advisor
- ChatGPT/Claude: General AI, not car-specific, no structured comparison

AutoMatch AI differs by combining: structured preference capture + AI advisor +
car list extraction + head-to-head comparison with colour-coded specs.

--------------------------------------------------------------------------------
3.1.4 Essential Features, Limitations, Requirements and Success Criteria
--------------------------------------------------------------------------------

Essential features:
- Preferences form: budget (with monthly/full choice), car type, main use,
  priorities, optional notes
- Validation: minimum 3 preferences selected before match
- AI recommendations: initial 3 cars with explanations
- Multi-turn chat: user can ask follow-up questions, AI maintains context
- Car list sidebar: extracts mentioned cars from chat, chronological order
- User selection: tick up to 14 cars as "good options"
- Compare: minimum 4 cars selected, fetch specs via API, compare head-to-head
- Tournament: 2 cars at a time, user picks winner, eliminations until 1 remains
- Stat colouring: green = better, red = worse (price lower, mpg higher, etc.)
- Persistence: save/restore chat and context in localStorage

Limitations:
- AI-generated specs may not match real-world data exactly
- No user accounts; data stored only in browser
- Requires internet and Groq API key (server-side)

Requirements:
- Modern browser (Chrome, Firefox, Safari, Edge)
- JavaScript enabled
- Deployed backend (e.g. Vercel) with GROQ_API_KEY set

Success criteria:
- User can enter preferences and receive initial AI recommendations
- User can chat to refine and get more car suggestions
- User can tick cars and compare 4–14 in head-to-head with specs
- User reaches a single "winner" car at the end of the compare flow

3.2.1 High Level Overview and Algorithms

SYSTEM OVERVIEW (Decomposition)

  AutoMatch AI
  ├── Preferences Module
  │   ├── Budget input (monthly/full)
  │   ├── Car type chips (multi-select)

  │   └── Notes textarea
  ├── Match Module
  │   ├── Build initial prompt
  │   ├── POST to /api/match
  │   └── Display AI response
  ├── Chat Module
  │   ├── Send user message
  │   ├── POST to /api/match (with history + context)
  │   ├── Stream/display AI reply
  │   └── Extract cars from chat
  ├── Car List Module
  │   ├── Parse car names (regex)
  │   ├── Display in chronological order
  │   └── Track selected (max 14)
  └── Compare Module
      ├── POST to /api/compare (with car names)
      ├── Tournament: pair up, show 2 at a time
      ├── User picks winner per pair
      └── Advance until 1 winner

DATA FLOW DIAGRAM

  User --> [Preferences] --> [Match API] --> [Groq AI] --> [Response]
    |                                                              |
    v                                                              v
  [Chat] <--> [Match API] <--> [Groq AI] <--> [Messages + Context]
    |
    v
  [Car List] --> [User picks 4–14] --> [Compare API] --> [Groq AI]
    |                                                              |
    v                                                              v
  [Tournament] <-- [JSON specs] <-- [Structured response] <--------

ALGORITHM: Extract Car Names (Pseudocode)

  FUNCTION extractCarsFromText(text):
      cars = empty set
      FOR each line IN split(text by newline):
          IF line matches "1. ..." OR "**...**:":
              name = extract name (strip **, strip description after ":")
              IF name length > 2 AND < 80:
                  ADD name to cars
      RETURN cars in order of first appearance

ALGORITHM: Tournament Elimination (Pseudocode)

  FUNCTION runTournament(cars):
      current = cars
      roundNum = 1
      WHILE current has more than 1 car:
          pairs = pairUp(current)  // [A,B], [C,D], or [E,null] for bye
          winners = empty array
          FOR each pair IN pairs:
              IF pair has 2 cars:
                  SHOW pair to user
                  winner = user picks one
                  ADD winner to winners
              ELSE:
                  ADD pair[0] to winners  // bye
          current = winners
          roundNum = roundNum + 1
      RETURN current[0]  // final winner

--------------------------------------------------------------------------------
3.2.2 Key Data (with validation)
--------------------------------------------------------------------------------

chatHistory (array)
  - Structure: [{ role: "user"|"assistant", content: string }, ...]
  - Validation: role must be "user" or "assistant"; content non-empty string

chatContext (object)
  - Fields: budget (string), budgetType ("monthly"|"full"), picks (array),
    notes (string)
  - Validation: picks must be array of strings; budgetType if set must be valid

selectedCarNames (Set)
  - Contents: car name strings
  - Validation: max 14 items; names must exist in getAllMentionedCars()

compareState (object)
  - Fields: cars (array of objects), roundNum (int), pairIdx (int), winners (array)
  - Validation: pairIdx < pairs.length; winners length matches pairs

API Request (match)
  - Initial: { budget, budgetType, picks, notes }
  - Chat: { messages: chatHistory, context: chatContext }
  - Validation: picks must have at least 3 for initial match

API Request (compare)
  - { cars: string[], context: chatContext }
  - Validation: cars must have 4–14 items

Iterative tests (during development):
  - Preferences form: select chips, verify summary updates
  - Match: submit with 3+ picks, verify AI response displays
  - Chat: send message, verify response and context preserved
  - Car list: verify cars extracted from different AI formats
  - Compare: select 4 cars, verify API returns specs, verify tournament flow

  - Test 4: Select 6 cars (odd) → expect bye in round 1
  - Test 5: Refresh page → expect restore modal; Continue → expect chat restored

--------------------------------------------------------------------------------

- Clear labels and hints on form (e.g. "helps the AI narrow down options")
